# `colorplay-new` feature parity decisions

## Decision vocabulary

- **Preserve verified:** keep the existing secure `colorplay` behavior as authority.
- **Rebuild:** retain the user need or interaction concept, then implement it inside the approved React/Supabase boundaries.
- **Compatibility only:** provide a safe link/configuration surface without importing external results.
- **Reject:** do not transfer the behavior, data, code, or branding.

No row authorizes implementation. The owning phase must receive its own approved plan.
