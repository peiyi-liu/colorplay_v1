# ColorPlay 專案章程與範圍

## 1. 文件目的

本文件定義 ColorPlay 的產品目標、範圍、角色、成功條件與非目標。所有後續設計與實作不得與本章程衝突。

## 2. 產品定位

ColorPlay 是「遊戲化課後複習與教師學習診斷平台」，不是單純選擇題網站，也不是以娛樂為主要目的的商業遊戲。

第一個正式內容領域為技術型高中設計群「色彩原理」，但資料模型與元件應保留未來擴展其他課程的能力。

### 核心學習循環

```text
進入班級／課程
→ 選擇章節與子主題
→ 閱讀微型複習內容
→ 進行互動挑戰
→ 即時正誤與解析
→ 獲得 XP／代幣／進度
→ 錯題補救與再次練習
→ 教師檢視班級與題目診斷
```

## 3. 產品目標

- P-001：學生能在手機、平板與電腦完成一致且可用的學習流程。
- P-002：所有正式作答、分數、XP、代幣與排行榜資料集中儲存於 Supabase PostgreSQL。
- P-003：學生無法透過修改前端 state、localStorage 或 API payload 任意增加正式獎勵或竄改成績。
- P-004：教師可建立與管理班級、內容、題庫及查看診斷資料。
- P-005：研究者可匯出具時間戳、題號、答案、作答時間與版本資訊的資料。
- P-006：平台具備可重複執行的自動測試與真實瀏覽器證據鏈。

## 4. 使用者角色

### Student

- 加入被授權的班級。
- 閱讀已發布內容。
- 建立並完成 quiz session。
- 查看自己的歷程、錯題、XP、代幣、頭像與可見排行榜。
- 不可查看正確答案原始欄位、其他學生私密紀錄或教師功能。

### Teacher

- 建立與管理自己擁有或被指派的班級。
- 建立、編輯、匯入、驗證與發布學習內容。
- 查看其班級成員的彙總分析與被允許的個別紀錄。
- 匯出其班級資料。
- 不可管理不屬於自己的班級。

### Platform Admin

MVP 僅保留最低限度管理能力，主要透過受控後台或 Supabase 管理操作。若建立應用內 Admin UI，需另立規格與 ADR，不得把 teacher 自動視為全域 admin。

## 5. MVP 必須包含

- Supabase Auth 登入、登出、session 恢復與角色辨識。
- 學生、教師角色與班級 membership。
- 六章色彩原理目錄結構，可只發布有內容章節。
- 複習卡。
- 單選題 quiz：隨機題序、20 秒計時、即時結果、解析、錯題摘要。
- 伺服器端答案判定、Quiz Score、XP、Token 與錢包交易。
- Level 與 Blook 頭像商店。
- 班級排行榜。
- 教師題庫 CRUD、批次匯入、發布狀態與驗證報告。
- 教師 dashboard：作答人次、平均正確率、錯誤率最高子主題、題目診斷。
- CSV／XLSX 研究資料匯出。
- RLS、審計、錯誤監控、備份與驗收證據。

## 6. MVP 非目標

以下不應阻塞 MVP：

- 即時多人同步對戰引擎。
- 語音、聊天、社群貼文。
- AI 自動出題或自動評分。
- 付費、訂閱或商業交易。
- Native iOS／Android App。
- SCORM／LTI 完整整合。
- 複雜 3D 遊戲。
- 全校級排課與教務系統。

Kahoot 可保留為外部連結工具，但不得把外部連結描述成 ColorPlay 內建多人引擎，也不得將 Kahoot 成績視為已同步，除非另行實作正式整合。

## 7. 後續版本方向

- V1.1：拖放配對、色彩排序、色票辨識。
- V1.2：色光／色料混色模擬、配色任務。
- V1.3：自適應題目推薦與弱點補救。
- V2：即時課堂競賽與房間狀態同步。

任何後續功能必須先有獨立 design spec 與 acceptance criteria。

## 8. 產品成功指標

MVP 上線後的產品指標至少包含：

- 核心學習流程完成率 ≥ 90%。
- 已開始 quiz session 中，可成功完成並寫入結果者 ≥ 98%。
- 正式環境資料寫入錯誤率 < 0.5%。
- 學生在第一次使用、無口頭協助下完成「登入→複習→作答→結果」中位時間 ≤ 8 分鐘。
- 核心頁面 Accessibility Lighthouse ≥ 95。
- 研究資料匯出與資料庫抽查筆數一致率 = 100%。

上述產品指標是上線後監控目標；程式驗收門檻見 `../acceptance/ACCEPTANCE_CRITERIA.md`。

## 9. 核心假設

- 學校或使用者具備現代瀏覽器與基本網路連線。
- 題庫與教學內容由教師或研究者負責審核。
- 學生可能使用共享裝置，因此 session 與登出流程必須清楚。
- 正式研究使用前，需完成研究倫理、告知同意與資料保存政策；平台功能不能取代法定程序。

## 10. 重要決策

- 內容與 UX 延續既有 ColorPlay 原型的品牌方向，但程式碼重新架構。
- PostgreSQL 是資料真實來源；localStorage 不是成績資料庫。
- 瀏覽器可直接使用 Supabase publishable／anon key，但所有 exposed tables 必須 RLS default-deny。
- 高權限 service role 只可出現在受控 server／Edge Function 環境。
- 正式驗收以實際 UI、資料庫狀態與可重現證據共同判定。
